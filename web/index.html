Sistem nasıl çalışıyor?

Tool call'u 100% biz çalıştırıyoruz. api.airforce gateway'i resmi olarak hiçbir model için tool calling desteklemiyor (panel'deki "Function Calling" filtresi 0 model döndürüyor). Bu yüzden tüm akışı bu köprü uniform şekilde yürütüyor:
  1. İstemci tools listesini bu köprüye gönderir.
  2. Köprü tools'u request'ten çıkarır; yerine system prompt'a <tool_calls> XML şeması enjekte eder.
  3. Upstream model XML formatında cevap verir (text içinde).
  4. Köprü XML'i parse eder, OpenAI/Anthropic native tool_calls / tool_use formatına çevirir.
  5. İstemci normal tool akışı görmüş gibi devam eder.
Upstream'in ürettiği "tool_calls" benzeri JSON field'lerine güvenmiyoruz — gateway resmi destek vermediği için tutarsız ya da yanıltıcı olabiliyor. Köprü her zaman text + XML'i kanon kabul eder.

Tier sınıflandırması (multiplier'a göre):

  • free multiplier ≤ 1 — ücretsiz hesapta sınırsız çalışır (rate-limit dakikada toplam multiplier bütçesi kadar). Probe önce bunlar için yapılır.
  • premium multiplier 2–10 — paralı ama Pay-As-You-Go değil; aboneliğe dahil olabilir. Free probe'ta atlanır (PROBE_TIER=premium ile dahil olur).
  • p2g multiplier > 10 — Pay-As-You-Go, kredi gerektirir. PROBE_TIER=all ile dahil olur.

Probe sonuç durumları:

  • capable Bridge bu modelde tool call'u çalıştırabiliyor. Probe sırasında system prompt'a XML şeması enjekte edildi, model doğru formatta tool çağrısı üretti, parse başarılı. /v1/models'da gösterilir.
  • incompat Model XML şemasını anlamıyor, boş döndürüyor, "tool yok" diyor, ya da timeout/429 alıyor. Bridge bu modeli işe alamıyor — listede gösterilmez.
  • payg Pay-As-You-Go / premium — free hesapta kullanılamıyor (gateway "Pay-As-You-Go" mesajı dönüyor). Listede gizli.
  • skipped Probe filtresine takıldı (image-only, midjourney, suno, vb. — chat dışı veya paid suffix'li modeller).
  • ✓ "XML" kolonu — son probe'ta XML akışı bu model için canlı doğrulandı.
  • – XML akışı bu modelde çalışmadı.
